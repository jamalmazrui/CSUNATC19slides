Contrast is King

Pastels... Pastels Everywhere!

•    We had a huge color palette problem

 

 

So, we dumped it…

•    We changed our entire branding and color palette

•     All text is 4.5:1 or higher

•     Most functional graphics are 3:1 or higher

 

•    But there’s one thing most products overlook

 

 

 

 

 

Text Selection / Highlight Contrast

•    4.5:1 between Text and Selection Color

•    3:1 between Selection Color and Background

 

 

 

 

 

 

Making Highlights Heard

Marking Up Highlights

•    There’s a tag for that! Meet <mark>

•    It’s the semantically correct way to mark up highlights

•     No screen reader support

•     Add clues manually

•     Buggy High Contrast Mode Support

•     Add High Contrast specific styles

<mark>

  <span class="visually_hidden">(highlight start)</span>

   search

  <span class="visually_hidden">(highlight end)</span>

</mark>

 

 

 

Highlighting Text

•    “Highlight Mode” is needed for keyboard users

•    Pressing arrow keys cause a page turn

•     In “Highlight Mode” page turns are disabled

•    Allow users to select with keyboard

•     Don’t forget the Caret Browsing (F7)

 

Mind The High Contrast

 

 

Windows High Contrast Mode

•    Often overlooked, but an important Assistive Technology

•    It’s not just inverted colors

•    Testing required (Real or Virtual Windows)

 

•    Pro Tip: Use High Contrast Media Queries

•    Pro Tip: Use Transparent Borders

 

 

 

 

 

 

Windows High Contrast Mode

@media screen and (-ms-high-contrast: active) {    

/* Styles in this section will only be applied when High Contrast Mode is activated.. Sadly only works in IE */  

border: solid 1px;

} 

    

.highContrastBorder{ 

   /* Transparent borders become visible when High Contrast Mode is activated */  

   border: solid 1px transparent;  

} 

 

 

 

 

 

 

Turning Pages

Turning Pages Is A Hassle

•    We can’t automate it (yet), but we can make it easier

•    Announce content on page change

•     Use a live region

•    Make sure page numbers are announced

•    Make sure progress is announced

•    Avoid multiple live regions

What Our Users Said?

Testers and Methodology

•    Five participants who frequently read digital books

•     Three Screen Reader (Blind)

•     One High Contrast / Magnifier (Low Vision)

•     One Keyboard-Only (Mobility/Dexterity)

•    No Timing

•    8 Tasks, focused on the reader component

 

What They Liked The Most…

•    Ability to resize text

•    Consistent page numbers

•    Content is easy to read

•    UI is straightforward and easy to explore

•    Vertical Scroll mode

•     Once they discovered it…

 

 

What Didn’t Work

•    Search has too many moving parts…

•    Several participants weren’t sure if they were taken to the correct place when performing a search.

 

 

What Didn’t Work

•    Skipping to a page number

•    Several participants used the previous/next links to jump to a different part of the book.

•    Provide a method to jump to a particular page

 

 

 

TL;DL Checklist…

•    Text and controls have adequate color contrast when different themes are applied

•    Selected and highlighted text have adequate contrast

•    Text and controls are visible and operable in Windows High Contrast Mode

•    There’s an option to view a complete section with vertical scroll

•    There’s an option to view text aligned to one side

•    There’s an option to enlarge text

TL;DL Checklist…

•    Book title, current section and current page information is always visible, and can be located by assistive technology users easily

•    Headings are correctly generated from the content and logically nested

•    Focus remains visible when using a keyboard

•    Page content is sanitized for Assistive Technology

•    Text flow and page number generation can adapt to changes in font size, including when a custom user style is applied
