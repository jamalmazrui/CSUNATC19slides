Involving people with disabilities

Bringing people with disabilities into every stage of the development process is incredibly important.

•              Validates the problem

•              Encourages feelings of ownership

•              Emphasize the importance of accessibility as a core user requirement

Design

Map out Tab Navigation (Interactive elements)

Be sure to map out your tab order, show where arrows are to be used, and build in keyboard shortcuts for critical commands.

 

Screen reader: Name, role, and value

Blind users need these three things to identify, understand and use the controls in your experience. Use this format:

Name. The visible label of the specific control, such as “Save” or “file list”

or an added name if the control has no visible text name

Role. The type of control, such as “Button”, or ”Checkbox”. This is the ARIA “role”

Value. The interactive state, such as “Disabled” or “Selected”

In rare cases, a description can be added if needed to understand the control. MUST be brief if present. Only needed if a control is non-standard or made of up other controls
