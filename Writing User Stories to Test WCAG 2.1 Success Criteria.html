WCAG Layers

  • Principles - perceivable, operable, understandable, and robust
  • Guidelines - basic goals to make content more accessible - not testable
  • Success Criteria - testable success criteria & more

    WCAG

    1. Perceivable Information and user interface components must be presentable to users in ways they can perceive.
    Guideline 1.1 Text Alternatives
    Provide text alternatives for any non-text content so that it can be changed into other forms people need, such as large print, braille, speech, symbols or simpler language.
    Success Criteria
    Success Criterion 1.1.1 Non-text Content
    Understanding > Benefits

    WCAG User Stories

    “As a (Benefits), I want (Success Criteria) so that (Value that relates to your product)”

    Sucess Criteria for Stories

    User stories aren't testable
    Behavior Driven Development (BDD) Criteria are testable
    GIVEN Exact context
    WHEN Action/Event
    THEN Outcomes
    AND/EXCEPT More of the same…
    GIVEN a non-text element exists
    WHEN a user's browser or AT focus in on the non-text elements
    THEN text alternative that serves the equivalent purpose are available
