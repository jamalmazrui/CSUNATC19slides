The Challenge

How do you train your site production staff to plan, design, develop and execute an accessible user experience with limited resources?

The Response

Start by empowering production leads to be accessibility experts within their chosen disciplines.

•       Grow organization core competence in digital accessibility over time

Requirements, best practices, bad ideas & failures

Must
“All img elements must have an alt attribute.”

Should
“Extended image descriptions should be available for all users.”

May
“Decorative images may be implemented via CSS backgrounds.”

Should not
“Alt text should not exceed 125 characters.”

Must not
“Decorative inline images must not contain descriptive alt text.”

Reinforce learning through teaching

•   Ambassadors co-present live training sessions with the Accessibility SME.

•   Teaching a subject requires Ambassadors to learn the web accessibility topic inside-out.

•   When Ambassadors teach their colleagues it helps to focus trainees on learning.

•   Taking on the responsibility of building and delivering the accessibility training fosters issue ownership in each Ambassador.

Humanize accessibility by connecting the “why” with the “how”

Match requirements with user benefits

Examples of requirements and best practices paired with positive user outcomes

For screen reader users who can’t see

•    Requirement: Informative, non-redundant inline images must have descriptive alt text values

•    Benefit: Screen reader users can hear the purpose or function of web page images


For voice recognition software users

•    Best practice: All page controls should have visible text labels

•    Benefit: Voice recognition users can quickly interact with page controls by announcing their names aloud

Foster empathetic creativity

•    Web accessibility is all about understanding the user experience for people with disabilities.

•    It is easier for trainees to commit to learning how to implement the technical details related to making content accessible if they understand why real people need the accommodation.

•    Opening Ambassadors’ minds to new perspectives creates opportunities to exercise empathetic creativity, a thought process required to craft user experiences that support the wide spectrum of needs and abilities in the disability community.

Accessibility Personas

Humanize the need for accessibility

Accessibility Persona – Vivian Colt

Vivian, 58, is a user who is sighted and who has severe arthritis.  She mostly interacts with web pages using a keyboard.

Vivian cannot operate a mouse for long due to pain and dexterity issues.

Vivian relies upon visual focus indicators to let her know where her keyboard focus is at all times.  For Vivian, a missing visual focus indicator is like a missing cursor for a mouse user.

Training example: Styling a visual focus indicator

Focus indicator was reset in our utilities.css file:

    color: #0e568c;

}

 

 

Possible Fix:

 

a.co_genericCollapse:focus,

a.co_tocImageLink:focus,

{

    outline-width: 1px;

}

 

Accessibility Persona – Jason Witt

Jason, 26, a user who is blind, interacts with a keyboard in tandem with JAWS screen reader software. Jason listens to web pages using JAWS software and headphones.

Jason “scans” pages with JAWS by pulling up a list of links on the page and listening to their announcement in isolation from the rest of the page content.  If links are not descriptively labeled, Jason doesn’t know which ones target the information he is seeking on the page.

Training example: aria-labelledby enhances link text

Accessibility Ambassador deliverables: training, job aids, wikis & internal standards

Job aid example: Accessible name decision tree

•                Decide how the accessible name of the content/control should be announced by a screen reader program.

•                Does old school HTML content and markup allow for the content/control to be announced correctly?  If yes, stop and use HTML markup for the accessible name. If no, proceed to Step 3.

•                Is there enough onscreen text to cobble together the desired accessible name for the content/control?  If yes, proceed to Step 4. If no, proceed to Step 5.

•                Use aria-labelledby on the element for the content/control and provide a space-separated list of id values associated with onscreen content. You are done!

•                Use aria-label to provide an accessible name, making sure the visible text label is contained within in the aria-label value. You are done!
