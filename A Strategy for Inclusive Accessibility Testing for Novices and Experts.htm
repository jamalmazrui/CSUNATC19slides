Objectives

•         Define manual accessibility testing and its importance within an accessibility effort

•         Identify challenges that manual testers face

Why manual testing?

•         Provides usability feedback unavailable through automated scanning of code

•         Validates keyboard/gesture usage and interaction when assistive technology is enabled

•         Provides springboard for early adoption of accessibility best practices into a development effort

•         Offers prospective for developers of digital content and their disabled audience

Assistive technology testing
for sighted testers

Screenreaders are rich in features for end users––many of which are not useful for sighted testers

•             Steep learning curve for operating and configuring

•             Many keyboard shortcuts and gestures

•             Manually file bugs with only voice output

•             Takes time to get consistent, reliable results

•             Difficult to multi-task with speech

What is JAWS Inspect?

•         Vastly simplifies accessibility and JAWS compatibility testing

•         Removes the distraction of speech or the complex feature set of the end-user product

•         Uses transcripts of JAWS output to quickly diagnose issues and share them easily across quality control and compliance systems

Benefits

•             Simplifies screenreader use for testers

•             Text transcript of JAWS output for use in bug tracking and compliance programs

•             Real-time identification of issues

•             Rapid rollout with limited training required

•             Demonstrates impact of accessibility on users

Launching JAWS Inspect

•             JAWS Inspect is a standard Windows application

•             Launch it from your Start menu or a program shortcut on your desktop

Using JAWS Inspect

•             The only persistence of JAWS Inspect you will see is a system tray icon

•             To launch JAWS Inspect, hold down your CTRL key and right-click the mouse (or use the context menu key)

•             The JAWS Inspect menu will appear on the current page in your browser

FAQs

•             Does JAWS Inspect require JAWS to be installed or running on the machine?

•          No, it is a completely standalone application

•          Which browsers does JAWS Inspect support?

•          Internet Explorer 11, Chrome, or Firefox

•          Which operating systems are supported?

•          Effectively the same as JAWS, so Windows based operating systems

•          Does JAWS Inspect identify compliance failures?

•          JAWS Inspect will provide a report of what JAWS users will experience on your site, but does not identify code level issues.
